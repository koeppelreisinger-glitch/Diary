# 作品申报补充材料

## 一、作品安装说明

1. 准备运行环境：安装 Python 3.11 及以上版本，并准备 PostgreSQL 数据库。

2. 进入项目根目录，创建并启用虚拟环境：

```powershell
python -m venv .venv
.venv\Scripts\activate
```

3. 安装项目依赖：

```powershell
pip install -r requirements.txt
```

4. 配置环境变量，在项目根目录创建 `.env` 文件，填写数据库连接、系统密钥和 AI 接口配置：

```env
DATABASE_URL=数据库连接地址
SECRET_KEY=安全随机字符串
TOKENHUB_AUTHORIZATION=AI授权信息
TOKENHUB_API_KEY=AI接口Key
TOKENHUB_MODEL=glm-4-flash
```

5. 初始化数据库：

```powershell
python init_db.py
```

6. 启动后端服务：

```powershell
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

7. 浏览器访问：

```text
http://localhost:8000/frontend/login.html
```

即可注册、登录并使用 Echo AI 日记应用。

## 二、设计思路

本作品以“降低日记记录门槛”为核心设计目标。传统日记需要用户主动组织语言，容易产生压力，难以长期坚持。因此，本作品将日记输入方式设计为 AI 对话，用户只需像聊天一样输入当天发生的事情，系统再通过 AI 自动整理成完整日记。

整体设计采用前后端分离思路。前端负责登录、今日记录、回忆、寻迹和个人中心等页面展示；后端使用 FastAPI 提供接口服务；数据库使用 PostgreSQL 保存用户、会话、日记和结构化信息；AI 模块负责对话回复、日记生成和内容提取。

系统不仅保存日记正文，还会自动提取事件、情绪、消费、地点和灵感等信息，使日记从普通文本升级为可搜索、可分析、可回顾的个人记忆数据。

## 三、设计重点难点

1. AI 对话与日记生成

重点是让用户通过自然聊天完成记录，并由 AI 自动整理出完整日记。难点在于 AI 输出内容需要稳定、清晰，并能符合系统需要的结构化格式。

2. 结构化数据提取

系统需要从用户的自然语言中提取事件、情绪、消费、地点、灵感等信息。难点在于用户表达方式多样，AI 返回结果可能不完全规范，因此后端需要进行解析、校验和兜底处理。

3. 数据库结构设计

本作品采用“日记主表 + 五类结构化子表”的设计。重点是既保存完整日记，又支持后续按不同维度查询。难点是要保证数据关系清晰，避免重复、混乱和越权访问。

4. 用户体验设计

作品需要让普通用户快速上手，因此前端界面尽量简洁，保留注册登录、今日记录、回忆和搜索等核心操作。难点是要在功能完整和使用简单之间取得平衡。

5. 安全与稳定性

系统涉及用户个人记录，需要保证登录鉴权、密码加密和用户数据隔离。难点是所有历史记录、图片和结构化数据都必须绑定当前用户，防止越权访问。
