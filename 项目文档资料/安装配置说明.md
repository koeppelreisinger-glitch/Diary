# Echo AI 日记应用安装配置说明

## 1. 环境要求

运行本项目需要准备以下环境：

| 环境 | 建议版本 | 说明 |
|---|---|---|
| Python | 3.11 或以上 | 后端运行环境 |
| PostgreSQL | 14 或以上 | 数据库存储 |
| pip | 随 Python 安装 | 安装 Python 依赖 |
| Git | 任意较新版本 | 获取和管理项目代码 |
| 浏览器 | Chrome / Edge 等 | 访问前端页面 |
| Vercel CLI | 可选 | 生产部署使用 |

## 2. 项目依赖

主要依赖位于 `requirements.txt`，包括：

- `fastapi`：后端 Web 框架
- `uvicorn`：本地运行服务
- `SQLAlchemy`：数据库 ORM
- `asyncpg`：PostgreSQL 异步驱动
- `pydantic`：数据校验
- `pydantic-settings`：环境变量配置
- `python-jose`：JWT 登录鉴权
- `bcrypt`：密码加密
- `httpx`：调用 AI 接口
- `Pillow`：图片处理

## 3. 本地安装步骤

### 3.1 进入项目目录

```powershell
cd "d:\antigravity files - 副本"
```

### 3.2 创建并激活虚拟环境

如果项目中已有 `.venv`，可以直接激活：

```powershell
.venv\Scripts\activate
```

如果没有虚拟环境，可先创建：

```powershell
python -m venv .venv
.venv\Scripts\activate
```

### 3.3 安装依赖

```powershell
pip install -r requirements.txt
```

## 4. 环境变量配置

项目需要在根目录创建 `.env` 文件。可参考 `.env.example`。

基础配置示例：

```env
DATABASE_URL=你的数据库连接地址
SECRET_KEY=一个足够安全的随机字符串
TOKENHUB_AUTHORIZATION=你的 AI 授权信息
TOKENHUB_API_KEY=你的 AI Key
TOKENHUB_MODEL=glm-4-flash
```

字段说明：

| 字段 | 说明 |
|---|---|
| `DATABASE_URL` | PostgreSQL 数据库连接地址 |
| `SECRET_KEY` | JWT 签名密钥，生产环境必须使用安全随机值 |
| `TOKENHUB_AUTHORIZATION` | TokenHub AI 授权信息 |
| `TOKENHUB_API_KEY` | TokenHub AI Key |
| `TOKENHUB_MODEL` | 使用的 AI 模型名称 |

注意：

- 如果 AI 配置缺失，AI 生成效果可能不可用或只能使用兜底逻辑。
- 生产环境不要使用默认 `SECRET_KEY`。
- 数据库连接必须保证后端服务可以访问。

## 5. 初始化数据库

首次运行项目时，需要初始化数据库表：

```powershell
python init_db.py
```

如果已有数据库结构，可根据实际情况执行项目中的迁移脚本，例如：

```powershell
python migrate_add_body_text.py
python migrate_add_image_url.py
python migrate_tags_to_inspirations.py
```

是否需要执行迁移脚本，取决于当前数据库是否已经包含对应字段和表结构。

## 6. 启动本地服务

在项目根目录运行：

```powershell
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

启动成功后可访问：

| 页面 | 地址 |
|---|---|
| 正式登录页 | `http://localhost:8000/frontend/login.html` |
| 开发调试入口 | `http://localhost:8000/frontend/debug.html` |
| 后端联调台 | `http://localhost:8000/frontend/test-console.html` |
| 健康检查 | `http://localhost:8000/health` |
| API 文档 | `http://localhost:8000/docs` |

## 7. Vercel 部署说明

项目已包含 `vercel.json`，支持将 FastAPI 后端和前端静态页面部署到 Vercel。

部署前需要在 Vercel 项目中配置环境变量：

```text
DATABASE_URL
SECRET_KEY
TOKENHUB_AUTHORIZATION
TOKENHUB_API_KEY
TOKENHUB_MODEL
```

部署命令示例：

```powershell
vercel --prod
```

注意：

- Vercel Serverless 环境不适合长期保存本地上传文件。
- 图片线上持久化建议后续接入对象存储，例如 S3、COS、Cloudinary 或 Supabase Storage。
- 数据库建议使用云 PostgreSQL，例如 Neon 或 Supabase，并配置可公网访问的连接地址。

## 8. 常见问题

### 8.1 页面打开后跳回登录页

通常是浏览器没有登录 Token，先访问：

```text
http://localhost:8000/frontend/login.html
```

完成注册或登录后再进入其他页面。

### 8.2 结束记录后没有马上生成日记

结束记录后，AI 生成在后台执行。可等待几秒后刷新，或调用：

```text
GET /api/v1/daily-records/today
```

如果返回 `is_generating=true`，说明还在生成中。

### 8.3 注册或登录失败

请检查：

- 手机号是否已注册。
- 密码是否符合长度要求。
- 数据库连接是否正常。
- 后端服务是否启动。

### 8.4 AI 功能不可用

请检查：

- `.env` 是否配置 `TOKENHUB_AUTHORIZATION` 或 `TOKENHUB_API_KEY`。
- AI Key 是否有效。
- 后端日志中是否有 AI 调用错误。

### 8.5 图片本地可用，线上不可用

本地环境可使用 `/uploads` 目录保存图片，但 Vercel 环境不提供稳定的持久化文件系统。线上图片功能建议接入对象存储。
