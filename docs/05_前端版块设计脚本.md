  /* 模拟水面：顶浅底深的蓝绿渐变，用现有 --grad-sky 衍生 */

### JS 渲染逻辑
```javascript
function renderEventStream(items) {
  const container = document.getElementById('xjStream');
  // 清除旧内容（保留伪元素水波）
  container.querySelectorAll('.xj-float-word').forEach(el => el.remove());

  if (!items || items.length === 0) {
    container.innerHTML += '<div class="xj-stream-empty">还没有事件记录</div>';
    return;
  }

  // 最多取 15 条事件，截断过长的文字
  const words = items.slice(0, 15).map(e => {
    const text = e.content.length > 18 ? e.content.slice(0, 18) + '…' : e.content;
    return text;
  });

  words.forEach((text, i) => {
    const el = document.createElement('div');
    el.className = 'xj-float-word';
    el.textContent = text;

    // 随机化：起始X位置、漂浮速度、延迟、横向偏移
    const x = 10 + Math.random() * 70;   // 10% ~ 80% 宽度
    const dur = 7 + Math.random() * 6;   // 7s ~ 13s 一轮
    const delay = -(Math.random() * dur); // 负延迟让初始分布不集中
    const drift = (Math.random() - 0.5) * 30; // -15px ~ +15px 横向漂移

    el.style.cssText = `
      left: ${x}%;
      --dur: ${dur}s;
      --delay: ${delay}s;
      --drift: ${drift}px;
      animation-delay: ${delay}s;
      animation-duration: ${dur}s;
    `;
    container.appendChild(el);
  });
}
```

## 十、JS 架构（精简版）

```javascript
// ── 状态 ──────────────────────────────────────────
const XJ = {
  range: 'month',
  cache: {}
};

// ── 入口 ──────────────────────────────────────────
async function initTrace() {
  if (!requireAuth()) return;
  updateRangeLabel();
  await loadAll();
}

// ── 时间范围 ──────────────────────────────────────
function setRange(r) {
  XJ.range = r;
  document.querySelectorAll('.xj-range-tab').forEach(t => {
    t.classList.toggle('active', t.dataset.range === r);
  });
  updateRangeLabel();
  loadAll();
}

function getDateRange() {
  const now = new Date();
  const end = now.toISOString().slice(0, 10);
  let start;
  const r = XJ.range;
  if (r === 'week') {
    const d = new Date(now); d.setDate(d.getDate() - 6);
    start = d.toISOString().slice(0, 10);
  } else if (r === 'month') {
    start = `${now.getFullYear()}-${String(now.getMonth()+1).padStart(2,'0')}-01`;
  } else if (r === 'quarter') {
    const d = new Date(now); d.setMonth(d.getMonth() - 3);
    start = d.toISOString().slice(0, 10);
  } else {
    start = '2000-01-01';
  }
  return { start, end };
}

// ── 并行加载 ──────────────────────────────────────
async function loadAll() {
  const { start, end } = getDateRange();
  const qs = `start_date=${start}&end_date=${end}&page_size=100`;
  const now = new Date();
  const calQs = `year=${now.getFullYear()}&month=${now.getMonth()+1}`;

  const [expenses, emotions, events, locations, inspirations, calendar] =
    await Promise.allSettled([
      apiFetch(`/history/expenses?${qs}`),
      apiFetch(`/history/emotions?${qs}`),
      apiFetch(`/history/events?${qs}`),
      apiFetch(`/history/locations?${qs}`),
      apiFetch(`/history/inspirations?${qs}`),
      apiFetch(`/history/calendar?${calQs}`)
    ]);

  renderExpenses(ok(expenses));
  renderEmotions(ok(emotions), ok(calendar));
  renderEventStream(ok(events)?.records);
  renderLocations(ok(locations));
  renderInspirations(ok(inspirations));
}

const ok = res => res.status === 'fulfilled' ? res.value : null;

// ── countUp ──────────────────────────────────────
function countUp(el, target, prefix = '¥') {
  const t0 = performance.now();
  const dur = 700;
  const step = now => {
    const p = Math.min((now - t0) / dur, 1);
    const ease = 1 - (1 - p) ** 3;
    el.textContent = prefix + (target * ease).toFixed(2);
    if (p < 1) requestAnimationFrame(step);
  };
  requestAnimationFrame(step);
}
```

# 回忆版块日记化改造脚本

## 1. 版块定位

“回忆”版块不再承担结构化数据看板职责，而是作为用户的私人日记集。

它的核心目标是让用户重新读回某一天：读回当时写下的话、那天的情绪氛围、留下来的照片，以及一句能把人带回当时的记忆摘录。

与“寻迹”版块的分工：

- 回忆：日记正文、摘录、照片、情绪氛围、时间流。
- 寻迹：地点、消费、情绪趋势、事件检索、结构化聚合。

产品判断规则：

> 如果一个内容帮助用户“重新读回某一天”，放在回忆。
> 如果一个内容帮助用户“分析自己的生活模式”，放在寻迹。

## 2. 页面信息架构

回忆页从上到下由六个视觉区域组成：

1. 顶部记忆头图区
2. 记忆搜索
3. 最近一篇日记
4. 记忆摘录
5. 照片回忆
6. 日记时间流

页面不展示五表 Tab，不展示结构化表格，不强调情绪分数、地点统计或消费筛选。

## 2.1 视觉一致性与春日氛围

回忆页不单独启用一套新的主题色，而是在 Echo 现有视觉系统里做“春日化”。

必须沿用：

- 全站主色 `--accent`
- 全站文字色 `--text-primary` / `--text-secondary` / `--text-muted`
- 全站卡片背景 `--bg-card`
- 全站边框 `--border` / `--border-light`
- 全站按钮类 `.btn` / `.btn-primary`

春日、大自然、太阳感只作为局部氛围层出现：

- 头图区使用很淡的暖黄与浅绿渐变。
- 头图区标题为“回忆”，副标题为“重新读回某一天，听见当时的自己。”
- 板块标题左侧用浅绿色细边作为自然感点缀。
- 摘录卡片和图片容器只使用低透明度暖色/绿色背景，不大面积换皮。
- 图片本身承担主要色彩，UI 不额外堆叠浓烈色块。
- 界面文案中不直接写“春日”两个字，氛围通过颜色、光影、留白和图片体现。

最终观感应当像 Echo 里的一个“阳光角落”，而不是另一个产品。

## 3. 记忆搜索

搜索框用于搜索日记正文、摘要、关键词、备注和当天线索（事件、灵感、地点、消费描述）。

输入提示文案：

> 搜索一句话、一个人、一件小事……

搜索结果仍以日记预览形式展示，而不是列表表格。

搜索命中后的页面语气：

- 有结果：展示匹配到的日记时间流。
- 有结果时，在“找到 X 段和某关键词有关的记忆”后显示“点击查看”，进入 `search.html?q=关键词`。
- `search.html` 是回忆搜索结果页，使用日记预览卡片展示命中的日期、情绪氛围、记忆摘录、正文预览、关键词，并支持上一页/下一页。
- 无结果：提示“没有找到这段记忆，换一个词试试。”

搜索边界：

- 回忆页不提供事件、地点、消费、情绪强度等复杂筛选。
- 这些功能统一归到寻迹页。

## 4. 最近一篇日记

首屏重点展示最近一篇已生成日记，而不是今日结构化数据。

内容组成：

- 日期和星期
- 情绪氛围标签
- 记忆摘录
- 日记正文预览
- 关键词
- “继续阅读”入口

示例文案：

> 4月27日 星期一
> 慢慢恢复的一天
>
> “今天其实没有发生什么惊天动地的事，但我能感觉到自己在一点点回到状态。”
>
> 今天早上起得有点晚，出门的时候很匆忙，连早饭都没有好好吃。上午主要在处理手头的任务……

## 5. 记忆摘录

记忆摘录是回忆页的情感钩子。

系统从日记正文或摘要里选出一句最有画面感或情绪感的话，形成摘录流。

展示方式：

- 摘录句作为主内容。
- 下方展示日期和情绪氛围。
- 点击进入对应日记详情。

示例：

> “我好像不是没有进步，只是今天终于看见了一点。”
> 4月18日 / 有完成感

摘录生成规则：

1. 优先从 `body_text` 中按句号、问号、感叹号、换行切句。
2. 选择长度适中、信息完整的一句。
3. 若无 `body_text`，使用 `summary_text`。
4. 若句子太短，退化为摘要预览。

## 6. 情绪氛围

回忆页不直接展示“情绪 6/10”这类数字。

数字分数转成氛围标签：

- 1-2：有点低落
- 3-4：有点疲惫
- 5：平静的一天
- 6-7：慢慢恢复
- 8：有完成感
- 9-10：被点亮的一天

情绪趋势和数字分析留给寻迹页。

## 7. 照片回忆

用户在今日对话中输入的图片，是回忆页的重要内容。

照片回忆不做相册管理工具，而是作为日记的视觉补充：

- 展示用户上传过的图片。
- 按日期分组。
- 保留 AI caption 作为图片旁白。
- 点击图片可查看大图。

页面中包含两个照片入口：

1. 照片里的今天：展示历史上同月同日的照片。
2. 记忆相片：展示最近若干天的图片组。

设计语气：

> 有些记忆不是写下来的，是被拍下来的。

空状态文案：

> 还没有照片回忆。在今日记录里发一张图，它会和那天的日记一起留下。

## 8. 日记时间流

日记时间流替代原来的历史记录表格。

每条日记展示：

- 日期
- 情绪氛围
- 记忆摘录
- 正文/摘要预览
- 关键词

交互：

- 点击整条日记进入详情页。
- 分页保留，但文案使用“上一页 / 下一页”，不强调总条数统计。

## 9. 详情页关系

回忆页点击日记后进入既有 `detail.html`。

后续可继续改造详情页，使其顺序变为：

1. 日期
2. 情绪氛围
3. 记忆摘录
4. 完整正文
5. 关键词
6. 折叠的当天线索

当前阶段先完成回忆首页的日记化改造。
