# 测试联调跑通：从聊天到生成总结闭环

本文档带你完整游走一遍 **新建对话 -> 产生记录 -> 完结会话 -> 查看总结** 的数据流。

## 准备工作
确保服务器正在运行：
```bash
.venv\Scripts\uvicorn.exe app.main:app --host 127.0.0.1 --port 8000 --reload
```
打开浏览器访问：`http://127.0.0.1:8000/docs`
并在顶部 `Authorize` 按钮输入已注册账号（例如前期的 13900001111 手机号）完成授权。

---

## 联调核心步骤（4步）

### 第一步：创建今天会话
- **路径**：`POST /api/v1/conversations`
- **操作**：直接点击 `Execute`。
- **预期结果**：
  返回类似于下方的成功响应，状态确保是 `recording`，并请**复制保存这个 `id` 值**。
  ```json
  {
    "code": 20000,
    "message": "success",
    "data": {
      "id": "e456d9a4-1234-abcd-8f90-111122223333",
      "status": "recording",
      "record_date": "2024-06-01",
      "created_at": "2024-06-01T10:00:00Z"
    }
  }
  ```

### 第二步：发生聊天互动，传递有效数据源信息
- **路径**：`POST /api/v1/conversations/{conversation_id}/messages`
- **操作**：
  1. 将第一步的 `id` 填入 `conversation_id` 框。
  2. 在 `Request body` 中粘贴：
  ```json
  {
    "content_type": "text",
    "content": "今天很累，开了很多会，还买了一杯咖啡花了32元"
  }
  ```
  3. 点击 `Execute`。
- **预期结果**：
  同时返回你的输入内容 (`user_message`) 及 AI 虚拟的占位反馈 (`ai_message`)。这是总结词条拉取信息的核心。

### 第三步：完结对话，触发后台结构化运算入库
- **路径**：`POST /api/v1/conversations/{conversation_id}/complete`
- **操作**：
  保持上文那个 `conversation_id` 填入，直接 `Execute`。
- **预期结果**：
  服务端会先把会话置为 `completing`，并把总结生成放入后台任务。成功返回 `20000`，但 `daily_record` 可能是 `null`，这属于正常现象。随后通过第四步轮询今日记录，直到生成完成。

### 第四步：从日历首页拉取核实数据完整度
- **路径**：`GET /api/v1/daily-records/today`
- **操作**：直接 `Execute`。
- **预期结果**：
  返回带有今日综合数据的查询体，且 `has_record` 为 `true`，`is_generating` 为 `false`。
  内部的 `record` 对象里不仅有主文，还带出了 `events`, `emotions`, `expenses`, `locations`, `inspirations` 等丰富明细。至此闭环彻底通顺。
