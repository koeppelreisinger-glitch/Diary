## 概述

本文档描述 Echo 日记应用「图片功能」的完整设计方案，涵盖：

1. **功能链路总览** — 图片从拍摄到呈现的完整数据流
2. **对话页改造** — 在聊天界面支持图片发送与展示
3. **后端架构** — 数据模型、存储方案、API 接口设计

图片功能是「回忆」版块从「数据查询」向「内容浏览」转型的核心支撑：
- **寻迹** 侧重聚合数据和趋势分析（情绪图谱、消费洞察等）
- **回忆** 侧重具体内容和叙事体验，**图片是内容维度最强的锚点**

## 一、功能链路总览

### 1.1 整体流程

```
用户在对话页点击 📷
        ↓
选择相册照片 / 拍摄新照片
        ↓
前端本地预览（可选择取消）
        ↓
点击「发送」
        ↓
① POST /api/v1/media/upload
  → 图片上传至存储（本地/OSS/S3）
  → 后端生成缩略图（异步）
  → 返回 { id, url, thumbnail_url }
        ↓
② 图片 URL 附在对话消息中发送
  POST /api/v1/conversations/{id}/messages
  Body: { content: "文字说明", image_id: 123 }
        ↓
③ AI 收到含图片的消息
  → 调用 Vision 模型（如 GPT-4o-vision / Gemini Vision）
  → AI 看图并生成回复
  → 同步将 ai_caption / ai_tags / dominant_colors 存回图片记录（异步）
        ↓
④ 图片绑定到当天的 DailyRecord
  → daily_record_images 表新增一行
  → record_date = 当天日期
        ↓
⑤ 回忆版块可按日期/图片流检索
  GET /api/v1/history/images
  GET /api/v1/history/images/{date}
  GET /api/v1/history/images/on-this-day
```

### 1.2 数据关系

```
users
  └── daily_records (一天一条)
        └── daily_record_images (一天可多张图)
              ├── 绑定到具体 conversation（可选）
              └── AI 分析字段（异步填充）
```

### 1.3 关键设计原则

| 原则 | 说明 |
|------|------|
| **图片归属于天** | 以 `record_date` 为核心索引，不依赖消息 ID |
| **AI 分析异步** | 上传后立即返回 URL，AI 分析在后台完成，避免阻塞 |
| **软删除** | 图片删除标记 `is_deleted=True`，不立即清除存储 |
| **缩略图分离** | 展示列表用缩略图，详情页用原图，减少流量消耗 |
| **用户隔离** | 所有查询严格绑定 `user_id`，不同用户图片完全隔离 |

```javascript
// 用户选择图片后触发
async function onImageSelected(event) {
    const file = event.target.files[0];
    if (!file) return;
    
    // 本地预览
    const objectUrl = URL.createObjectURL(file);
    document.getElementById('imgPreviewThumb').src = objectUrl;
    document.getElementById('imgPreviewArea').style.display = 'flex';
    
    // 上传到后端
    try {
        const formData = new FormData();
        formData.append('file', file);
        
        const resp = await fetch('/api/v1/media/upload', {
            method: 'POST',
            headers: { 'Authorization': `Bearer ${getToken()}` },
            body: formData
        });
        const json = await resp.json();
        _pendingImageId = json.data.id;  // 保存上传后的图片 ID
    } catch (e) {
        showToast('图片上传失败，请重试');
        clearImageSelection();
    }
}

// 取消选择图片
function clearImageSelection() {
    _pendingImageId = null;
    document.getElementById('imgFileInput').value = '';
    document.getElementById('imgPreviewArea').style.display = 'none';
}

// 发送消息（扩展现有逻辑）

        html += `<div class="bubble-text">${escapeHtml(text)}</div>`;
```

### 2.4 AI 调用方式（后端）

#### API 规范

| 项目 | 值 |
|------|----|
| **Endpoint** | `https://tokenhub.tencentmaas.com/v1/chat/completions` |
| **Model** | `glm-5v-turbo`（支持图片 + 文本多模态输入）|
| **认证** | `Authorization: Bearer sk-Xnsn1B0GejGc8B7I0P6gIQSshINdq45WClHbBc1WAHntLRde` |
| **对话回复** | `"stream": true`（流式，减少首字延迟）|
| **图片分析** | `"stream": false`（后台异步，一次性读取结果写库）|

#### 场景 A：含图片的对话消息（流式，同步路径）

当用户发送含 `image_id` 的消息时，后端构建 Vision 格式请求并以流式响应调用 AI。

```python
import httpx, json

AI_BASE_URL = "https://tokenhub.tencentmaas.com/v1"
AI_API_KEY  = "sk-Xnsn1B0GejGc8B7I0P6gIQSshINdq45WClHbBc1WAHntLRde"
AI_MODEL    = "glm-5v-turbo"

async def build_vision_user_message(message, session) -> dict:
    """构建含图片的用户消息（OpenAI Vision 格式）"""
    content = []
    if message.image_id:
        image = await get_image_by_id(message.image_id, session)
        content.append({
            "type": "image_url",
            "image_url": {"url": image.url, "detail": "low"}
        })
    if message.content:
        content.append({"type": "text", "text": message.content})
    # 若只有图片没有文字，补充引导语
    if message.image_id and not message.content:
        content.append({
            "type": "text",
            "text": "请描述这张图片，并结合我的日记风格给出温暖的回应。"
        })
    return {"role": "user", "content": content}


async def call_ai_vision_stream(history_messages: list, user_message: dict) -> str:
    """含图片的对话 AI 调用（流式）。在 POST /messages 同步路径中调用。"""
    payload = {
        "model": AI_MODEL,
        "messages": history_messages + [user_message],
        "stream": True,
        "max_tokens": 500
    }
    headers = {
        "Authorization": f"Bearer {AI_API_KEY}",
        "Content-Type": "application/json"
    }
    full_reply = ""
    async with httpx.AsyncClient(timeout=30.0) as client:
        async with client.stream(
            "POST", f"{AI_BASE_URL}/chat/completions",
            headers=headers, json=payload
        ) as response:
            response.raise_for_status()
            async for line in response.aiter_lines():
                if not line.startswith("data: "):
                    continue
                data_str = line[6:]
                if data_str.strip() == "[DONE]":
                    break
                try:
                    chunk = json.loads(data_str)
                    delta = chunk["choices"][0]["delta"].get("content", "")
                    if delta:
                        full_reply += delta
                except (json.JSONDecodeError, KeyError, IndexError):
                    continue
    return full_reply
```

```python
from sqlalchemy import Column, Integer, String, Boolean, DateTime, Date, ForeignKey
from sqlalchemy.sql import func
from app.db.base_class import Base

class DailyRecordImage(Base):
    """日记图片表 — 每张图片对应用户某天的记录"""
    
    __tablename__ = "daily_record_images"

    # ── 主键与外键 ──
    id = Column(Integer, primary_key=True, index=True)
    user_id = Column(Integer, ForeignKey("users.id"), nullable=False, index=True)
    daily_record_id = Column(Integer, ForeignKey("daily_records.id"), nullable=False)
    record_date = Column(Date, nullable=False, index=True)  # 冗余，便于日期查询

    # ── 存储信息 ──
    storage_key = Column(String(512), nullable=False)        # OSS/本地路径 Key
    url = Column(String(1024), nullable=False)               # 公开访问 URL
    thumbnail_url = Column(String(1024), nullable=True)      # 缩略图 URL（异步生成）

    # ── 文件元信息 ──
    original_filename = Column(String(255), nullable=True)
    mime_type = Column(String(64), nullable=False)           # image/jpeg / image/png
    file_size = Column(Integer, nullable=False)              # 字节数
    width = Column(Integer, nullable=True)                   # 像素宽度
    height = Column(Integer, nullable=True)                  # 像素高度

    # ── 来源追踪（可选）──
    conversation_id = Column(String(64), nullable=True)      # 发送时所在的对话 ID

    # ── AI 分析字段（异步后台填充）──
    ai_caption = Column(String(500), nullable=True)
    # 示例："温暖的书店内景，木质书架，暖黄色灯光，排列整齐的书籍"
    
    ai_tags = Column(String(500), nullable=True)
    # 存储 JSON 字符串：'["室内", "书店", "暖色调", "文艺", "书籍"]'
    
    dominant_colors = Column(String(200), nullable=True)
    # 存储 JSON 字符串：'["#F5D76E", "#8B4513", "#2C1B0E"]'
    # 取前3主色，用于回忆版块色调可视化

    # ── 软删除 ──
    is_deleted = Column(Boolean, default=False, nullable=False)

    # ── 时间戳 ──
    created_at = Column(DateTime(timezone=True), server_default=func.now())
    updated_at = Column(DateTime(timezone=True), onupdate=func.now())
```

### 3.4 Service 层核心逻辑

```python
# app/services/media_service.py（核心逻辑摘要）

class MediaService:

    @staticmethod
    async def upload_image(session, user_id: int, file: UploadFile):
        """
        上传图片流程：
        1. 验证文件类型和大小
        2. 生成唯一 storage_key（user_id/date/uuid.ext）
        3. 保存文件（本地开发 / 生产用 OSS）
        4. 读取图片尺寸（Pillow）
        5. 获取或创建当天 DailyRecord
        6. 插入 DailyRecordImage 记录
        7. 触发异步任务：生成缩略图 + AI 分析
        8. 返回 ImageUploadResponse
        """
        # 验证
        MAX_SIZE = 10 * 1024 * 1024  # 10MB
        ALLOWED_TYPES = {"image/jpeg", "image/png", "image/webp"}
        
        if file.content_type not in ALLOWED_TYPES:
            raise ErrorResponseAPIException(400, "不支持的图片格式", 40002)
        
        content = await file.read()
        if len(content) > MAX_SIZE:
            raise ErrorResponseAPIException(400, "图片大小不能超过 10MB", 40003)
        
        # 存储
        today = date.today()
        ext = file.content_type.split("/")[1]
        storage_key = f"images/{user_id}/{today}/{uuid4().hex}.{ext}"
        url = await StorageService.save(storage_key, content)
        
        # 读取尺寸
        from PIL import Image as PilImage
        import io
        pil_img = PilImage.open(io.BytesIO(content))
        width, height = pil_img.size
        
        # 获取或创建今日记录
        daily_record = await DailyRecordService.get_or_create_today(session, user_id)
        
        # 写入数据库
        img_record = DailyRecordImage(
            user_id=user_id,
            daily_record_id=daily_record.id,
            record_date=today,
            storage_key=storage_key,
            url=url,
            original_filename=file.filename,
            mime_type=file.content_type,
            file_size=len(content),
            width=width,
            height=height
        )
        session.add(img_record)
        await session.commit()
        await session.refresh(img_record)
        
        # 触发异步后台任务
        asyncio.create_task(
            MediaService._async_process_image(img_record.id, content)
        )
        
        return ImageUploadResponse(
            id=img_record.id,
            url=img_record.url,
            thumbnail_url=None,  # 异步生成，稍后可查
            width=width,
            height=height,
            file_size=len(content)
        )

    @staticmethod
    async def _async_process_image(image_id: int, content: bytes):
        """
        后台异步处理（非阻塞，图片上传后 create_task 触发）：
        1. 生成 300px 宽缩略图，上传并更新 thumbnail_url
        2. 调用 AI Vision（glm-5v-turbo，stream=false）获取 ai_caption / ai_tags
        3. 提取主色调（Pillow quantize）
        4. 一次性写回数据库

        AI API：POST https://tokenhub.tencentmaas.com/v1/chat/completions
        Model：glm-5v-turbo | stream：false（后台分析，无需流式）
        """
        import io, json, base64, asyncio, logging
        import httpx
        from PIL import Image as PilImage

        AI_URL     = "https://tokenhub.tencentmaas.com/v1/chat/completions"
        AI_KEY     = "sk-Xnsn1B0GejGc8B7I0P6gIQSshINdq45WClHbBc1WAHntLRde"
        AI_MODEL   = "glm-5v-turbo"
        logger     = logging.getLogger(__name__)

        try:
            pil_img = PilImage.open(io.BytesIO(content)).convert("RGB")

            # ── 步骤 1：生成缩略图 ──────────────────────────────
            thumb_w = 300
            thumb_h = int(pil_img.height * thumb_w / pil_img.width)
            thumb   = pil_img.resize((thumb_w, thumb_h), PilImage.LANCZOS)
            buf = io.BytesIO()
            thumb.save(buf, format="JPEG", quality=80)
            thumbnail_url = await StorageService.save(
                f"thumbnails/{image_id}.jpg", buf.getvalue()
            )

            # ── 步骤 2：提取主色调（CPU 密集，放线程池）──────────
            dominant_colors = await asyncio.get_event_loop().run_in_executor(
                None, MediaService._extract_dominant_colors, pil_img
            )

            # ── 步骤 3：调用 AI Vision 分析图片 ─────────────────
            b64 = base64.b64encode(content).decode()
            prompt = (
                '请用中文分析图片，严格按 JSON 返回，不要多余文字：\n'
                '{"caption":"一句话描述图片内容（≤50字）",'
                '"tags":["标签1","标签2","标签3","标签4","标签5"]}'
            )
            payload = {
                "model": AI_MODEL,
                "messages": [{
                    "role": "user",
                    "content": [
                        {"type": "image_url",
                         "image_url": {"url": f"data:image/jpeg;base64,{b64}",
                                       "detail": "low"}},
                        {"type": "text", "text": prompt}
                    ]
                }],
                "stream": False,   # 后台分析，非流式
                "max_tokens": 200,
                "temperature": 0.3
            }
            headers = {
                "Authorization": f"Bearer {AI_KEY}",
                "Content-Type": "application/json"
            }

            ai_caption, ai_tags = None, None
            async with httpx.AsyncClient(timeout=20.0) as client:
                resp = await client.post(AI_URL, headers=headers, json=payload)
                resp.raise_for_status()
                raw = resp.json()["choices"][0]["message"]["content"]
                try:
                    parsed    = json.loads(raw[raw.index("{"):raw.rindex("}") + 1])
                    ai_caption = parsed.get("caption")
                    ai_tags    = json.dumps(parsed.get("tags", []), ensure_ascii=False)
                except (ValueError, json.JSONDecodeError):
                    ai_caption = raw[:200]  # 兜底：直接截取原文

            # ── 步骤 4：写回数据库 ───────────────────────────────
            async with get_async_session() as session:
                await session.execute(
                    update(DailyRecordImage)
                    .where(DailyRecordImage.id == image_id)
                    .values(
                        thumbnail_url=thumbnail_url,
                        ai_caption=ai_caption,
                        ai_tags=ai_tags,
                        dominant_colors=json.dumps(dominant_colors, ensure_ascii=False)
                    )
                )
                await session.commit()

        except Exception as e:
            logger.error(f"[MediaService] 图片异步处理失败 image_id={image_id}: {e}")
            # 后台失败不影响用户侧功能，字段保持 NULL 即可

    @staticmethod
    def _extract_dominant_colors(img: "PilImage.Image", n: int = 3) -> list:
        """提取图片前 n 个主色，返回 hex 列表，如 ['#F5D76E', '#8B4513']。"""
        small     = img.resize((100, 100))
        quantized = small.quantize(colors=n * 3, method=2).convert("RGB")
        palette   = sorted(quantized.getcolors(10000) or [], key=lambda x: x[0], reverse=True)
        seen, result = set(), []
        for _, rgb in palette:
            h = "#{:02X}{:02X}{:02X}".format(*rgb)
            if h not in seen:
                seen.add(h)
                result.append(h)
            if len(result) >= n:
                break
        return result
```
